# ADK DevTools Agents

## What the project does

The `devtools` module within the `adk-agents` project provides a suite of specialized agents designed to automate and streamline various development tasks. These agents offer functionalities ranging from file system management to pipeline orchestration and README file generation, making routine development workflows more efficient.

## Why the project is useful

This project aims to enhance developer productivity by:

*   **Automating Repetitive Tasks**: Agents handle common development chores, freeing up developers to focus on core logic.
*   **Modular and Extensible Design**: Each agent is independently configurable and can be extended or integrated into existing workflows.
*   **Simplified File System Operations**: The `FileSystemAgent` provides programmatic access to file and directory manipulations.
*   **Workflow Automation**: The `PipelineAgent` facilitates the creation and management of automated pipelines.
*   **README Generation**: The `ReadmeAgent` assists in creating and maintaining project documentation.

## How users can get started

To get started with ADK DevTools Agents, follow these steps:

### Prerequisites

*   **Java Development Kit (JDK)**: Version 17 or higher.
*   **Apache Maven**: Version 3.6.3 or higher.
*   **Git**: For cloning the repository.

### Installation and Setup

1.  **Clone the repository**:

```bash
    git clone https://github.com/svetanis/adk-agents.git
    cd adk-agents
```

2.  **Build the project**:
    Navigate to the project root and build using Maven:

```bash
    mvn clean install
```

### Configuration

Agents are configured using YAML files located in the `src/main/resources/devtools` directory.

*   `filesystem-agent.yaml`: Configures the `FileSystemAgent`.
*   `readme-agent.yaml`: Configures the `ReadmeAgent`.

You can modify these files to customize agent behavior as needed. For example, to configure the `ReadmeAgent`:

```yaml
# src/main/resources/devtools/readme-agent.yaml
agentName: ReadmeAgent
outputFile: README.md
sections:
  - title: "Introduction"
    content: "This project provides tools for..."
  - title: "Usage"
    content: "To use this, run..."
```

### Usage Examples

The `DevToolsApp.java` acts as the main entry point to run the agents. You can run it from your IDE or via Maven.

**Running via Maven Exec Plugin (Example):**

After building the project, you can run the application. The exact command might vary depending on how `DevToolsApp` is designed to receive agent configurations. A common approach is to run the JAR file generated by Maven:

```bash
mvn compile exec:java -Dexec.mainClass=com.svetanis.agents.devtools.DevToolsApp
```

You would typically configure which agent to run, and with what parameters, either through command-line arguments (if `DevToolsApp` supports it) or by modifying the configuration files.
